=== Single source of truth

Each person is a file at `site/_fragments/people/<slug>.adoc`. Flat, no subfolders. The YAML frontmatter is the database. Body either has prose (complete) or `_Stub -- body to be written._` / `// FixMe: Captain` (stub).

=== Lean schema (current, post-cleanup)

[source,yaml,opts=novalidate]
----
---
name: ...                # canonical display string
born: ...                # signed int; negative for BCE; null/omitted for unknown
died: ...                # signed int; null/omitted for alive or unknown
era: ...                 # antiquity | medieval | early-modern | modern | contemporary (renaissance reserved)
nationality: ...         # short tag
sciences: [...]          # superset of all fields the person worked in
science: ...             # single primary field (directory column)
related: [...]           # slugs for "related people" breadcrumb (2-3 typical)
---
= Name (dates)
ifndef::env-site[:imagesdir: ../../assets/images/people]
ifdef::env-site[:imagesdir: /sindri-labs/assets/images/people]
:page-image: <filename>.jpg          # placeholder.jpg until real portrait arrives
:page-image-credit: "<attribution>"  # "change me" for placeholders

image::{page-image}[{page-image-credit},width="500px",align="left"]

<body or stub marker>
----

For *group fragments* (discovery pages, not portraits) the schema is even leaner — only the two superset fields:

[source,yaml]
----
---
sciences: [...]   # superset across members
related: [...]    # members + their externally-related people
---
----

For the *synopsis variant* (`democritus-synopsis`):

[source,yaml]
----
---
synopsis_of: democritus
---
----

=== Querying the data

[source,bash]
----
# All people whose primary science is physics:

grep -lE '^sciences:.*mathematics' site/_fragments/people/*.adoc

# Era counts:
awk '/^era:/ {print $2}' site/_fragments/people/*.adoc | sort | uniq -c

# Stubs (no body yet, FixMe marker or "Stub" placeholder):
